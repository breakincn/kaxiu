商户收款二维码 + 卡包直购

功能背景

目前大量小微商户（理发店、洗车店、美容店、澡堂、送水站等）：

没有完整的线上系统

主要依赖 支付宝 / 微信个人或商户收款码

线下售卖 次数卡 / 充值卡 时流程繁琐：

先介绍 → 再扫码付款 → 再人工登记 → 容易出错

卡包希望在不接入支付、不碰资金、不替商户收款的前提下，
帮助商户 把“卖卡 + 收款 + 建卡”流程一次完成。

功能描述（对外标准描述）

商户可在卡包商户端后台填写自己的 支付宝或微信收款号，
系统将为该商户生成一个专属的「卡包售卡二维码」。

用户使用卡包用户端扫码后：

自动进入该商户的 卡片售卖页面

查看当前在售的卡片（次数卡 / 充值卡 / 课时卡等）

选择卡片后，系统 跳转至商户自己的支付宝或微信完成付款

用户付款完成后，返回卡包并完成 卡片开卡/入包

整个过程中：

资金 直接进入商户的支付宝或微信

卡包 不参与收款、不代管资金

卡包仅负责 卡片展示、用户确认、卡片记录

商户端功能说明（后台）
1️⃣ 收款信息配置

商户可配置以下信息之一或多个：

支付宝收款码（图片或收款链接）

微信收款码（图片或收款链接）

说明：
卡包仅保存商户提供的收款信息，不校验、不代收、不分账。

2️⃣ 卡片售卖配置

商户可在后台配置：

卡片名称（如：洗车 10 次卡）

卡片类型（次数卡 / 充值卡 / 课时卡）

售价

总次数 / 充值金额

有效期（可选）

是否支持预约/排队（可选）

3️⃣ 生成售卡二维码

系统为每个商户生成一个 固定售卡二维码，用于：

线下张贴（前台、收银台）

朋友圈 / 群分享

海报 / 名片

二维码指向地址示例：

kabao.me/s/mitao

用户端流程说明
用户扫码后流程

打开卡包用户端（H5）

进入商户页面：

商户名称

当前在售卡片列表

选择卡片 → 点击「购买」

跳转至商户的支付宝 / 微信完成付款

付款完成后点击「已完成付款」

系统生成对应卡片，加入用户卡包

⚠️ 关键设计点
卡片生成以「用户确认付款完成」为准
卡包不校验真实到账金额

对商户的直接价值
不用开发系统，不换收款方式，也能把“卖卡”变成一次扫码完成的标准流程。
帮商户把「卖卡 + 收款 + 建卡」变成一次扫码完成，但钱永远直接进商户账户

对商户来说：

钱 → 直接进我自己的支付宝 / 微信

不走平台

不抽成（至少明面上）


非常适合线下场景:
前台贴一个二维码
桌上放一个二维码
墙上贴一个二维码

顾客：
扫 → 看卡 → 付钱 → 自动进卡包
不需要商户解释、不需要人工操作。
